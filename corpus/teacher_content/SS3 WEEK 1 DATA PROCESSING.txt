INDEXES
An index in a database is a data structure designed to enhance the speed of data retrieval operations. It provides a fast lookup mechanism, much like an index in a book that allows you to find information quickly without reading every page. Indexes are particularly useful for speeding up queries, sorting data, and joining tables. An index typically consists of two main columns:
Search Key: A copy of the primary or candidate key from the database table.
Data Reference: Pointers to the disk blocks where the actual data records corresponding to the search key are stored.
TYPES OF INDEXES
PRIMARY INDEX
A primary index is built on a table’s primary key, ensuring each entry uniquely corresponds to one row in the table. It determines the order of records on disk and is often a clustered index. The primary index contains records that are usually organized in sorted order.
Types:
Dense Index: Contains an index entry for every search key value in the database, making searches fast but requiring more storage space.
Sparse Index: Stores index entries for only some search key values, reducing storage and maintenance but may be slower for searches.
Characteristics:
Guarantees uniqueness of primary key values.
Often serves as a clustered index, though it does not have to be.

CLUSTERED INDEX
A clustered index determines the physical order of data in a table, meaning the rows are stored on disk in the same sequence as the index. Each table can have only one clustered index because the data can be sorted in only one way. Unlike non-clustered indexes, a clustered index organizes the actual data rows in the table itself, rather than just storing pointers to the data.
Characteristics:
Data is stored in the order of the clustered index key.
Often, the primary key serves as the clustered index, but it can be set on any column.
Insertions or updates may require rearranging the data to maintain the order, which can be resource-intensive.
UNCLUSTERED (NON-CLUSTERED/SECONDARY) INDEX
An unclustered index (also known as a non-clustered or secondary index) does not change the physical order of data in a table. Instead, it creates a separate structure with index entries that point to the actual data rows stored elsewhere on disk. Unlike a clustered index, it does not affect how the data is organized.
Characteristics:
Multiple unclustered indexes can exist per table.
It contains pointers to data rows, requiring an additional lookup to retrieve the actual data.
The physical arrangement of the table remains unchanged, allowing for flexible indexing.
Can include duplicate values, unlike primary indexes.
Provides alternative paths to access data, which can speed up queries on non-key columns.
