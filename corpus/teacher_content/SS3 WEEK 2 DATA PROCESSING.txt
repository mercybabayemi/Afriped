INDEXES
COMPOSITE INDEX (CONCATENATED KEYS)
A composite index (or concatenated key) is an index built on two or more columns of a table. It optimizes queries that filter or sort by multiple columns simultaneously. Composite indexes are useful for efficiently handling searches involving combinations of columns.
Characteristics:
The order of columns in the composite index is crucial for query optimization.
Ideal for queries that use multiple columns for filtering or sorting.
Enhances performance for complex queries involving multiple fields.
DUPLICATE INDEXES
Duplicate indexes occur when multiple entries in an index have the same key value, meaning several rows in the table share the same indexed value. This can lead to inefficiencies in index performance and storage.
Handling Duplicates:
Primary Indexes: Primary indexes are built on the primary key, which is unique by definition. Therefore, primary indexes do not allow duplicates as they enforce uniqueness in the table.
Secondary Indexes: Secondary indexes can contain duplicate values since they are not bound by the uniqueness constraint. However, you can create a unique secondary index to prevent duplicates if needed.
Examples of Handling Duplicates:
A non-unique secondary index on "LastName" might list multiple entries for "Smith," reflecting multiple people with the same last name.
A unique index on "Email" prevents duplicate entries, ensuring that each email address in the database is unique.

DIFFERENCES BETWEEN CLUSTERED INDEX AND NON- CLUSTERED INDEX
CLUSTERED INDEX
NON- CLUSTERED INDEX
Clustered index is faster.
Non-clustered index is slower.
Clustered index requires less memory for operations.
Non-Clustered index requires more memory for operations.
In clustered index, index is the main data.
In Non-Clustered index, index is the copy of data.
A table can have only one clustered index.
A table can have multiple non- clustered index.
Clustered index store pointers to block not data.
Non-Clustered index store both value and a pointer to actual row that holds data.
